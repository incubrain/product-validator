### Tags

- **{CONFIG}** â†’ Things the user must update when cloning.
- **{TASK}** â†’ Work that still needs doing.
- **{OPTIMIZE}** â†’ Things to consider after validation step
- **{DOC}** â†’ Areas needing documentation.
- **{DECISION}** â†’ Why something was implemented this way.
- **{DX}** â†’ Developer experience tips.
- **{FRICTION}** â†’ Things causing friction for developer setup or customer
  conversion

### Example

```js

// {TASK}: Add unit test for validation
// {OPTIMIZE}: Consider adding optimized, scripted hero video, may boost your conversions

// {DX}: use this keyboard shortcut to do this thing
// {FRICTION}: This file needs to be deleted before going live
```
